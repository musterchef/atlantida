Mapping Audiovisivo: Dati Meteo

Il meteo aggiunge un layer atmosferico che cambia lentamente — perfetto per modulare l'ambiente:

☁️ Nuvole → Opacità / Densità

cloud_norm (0→1): controlla la densità dello sfondo, nebbia volumetrica, opacità dei layer. Le 3 sotto-bande (cloud_low/mid/high_norm) possono guidare 3 layer di nebbia a profondità diverse.

🌡️ Temperatura → Palette Colore

temperature_norm: modula il colore globale. Freddo (basso) → blu/violetto. Caldo (alto) → arancio/rosso. apparent_temperature è più espressivo perché include il wind chill percepito in bici.

💨 Vento → Movimento / Turbolenza

wind_norm → velocità del noise/particelle. wind_gust_norm → impulsi di turbolenza. wind_dir_norm → direzione del flusso di particelle.

🌧️ Pioggia → Disturbo / Texture

precip_norm: aggiunge gocce, distorsione, rumore visivo. Triggerabile come evento: quando precip_norm > 0 → attiva un layer "pioggia".

🌫️ Visibilità → Profondità

visibility_norm: controlla la distanza di rendering, fade del background, densità della foschia. Valori bassi = atmosfera chiusa, misteriosa.

📊 Pressione → Drone / Tensione

pressure_norm: cambiamento lento, ideale per un drone di fondo che sale/scende impercettibilmente. Pressione bassa = temporale in arrivo → crescendo drammatico.

Idee di Composizione per Layer

Layer 1: Terreno (lento, costante)

Guidato da terrain_type + ele_norm. Cambia nell'ordine dei minuti. Definisce il "colore" di base della scena: morfologia, materiale visivo, timbro sonoro di fondo.

Layer 2: Atmosfera (lento, modulazione)

Guidato dai dati meteo: cloud_norm, temperature_norm, visibility_norm. Cambia gradualmente. Modula l'ambiente: luce, foschia, calore cromatico.

Layer 3: Cinematica (medio, ritmo)

Guidato da speed_kmh, slope, difficulty. Segue il ritmo della pedalata. Controlla energia, pulsazione, intensità.

Layer 4: Gesto (veloce, micro-modulazione)

Guidato da curvature, effort, bearing_deg. Variazioni rapide, curve, sforzi. Controlla glitch, deformazioni, accenti ritmici.

Layer 5: Eventi (trigger discreti)

Guidato da weather_code (cambi di condizione), POI (passaggio monumenti), precip_norm che passa da 0 a > 0. Trigger per cambi scena, flash, drop sonori.

Esempio: GLSL Shader con Dati Meteo

// Fragment shader — esempio di composizione atmosferica
// I valori arrivano come uniform da TD

uniform float cloud_norm;      // 0→1
uniform float temperature_norm; // 0→1
uniform float wind_norm;        // 0→1
uniform float precip_norm;      // 0→1
uniform float visibility_norm;  // 0→1
uniform float ele_norm;         // 0→1

void main() {
    vec2 uv = vUV.st;
    
    // Colore base: temperatura → palette freddo/caldo
    vec3 cold = vec3(0.15, 0.2, 0.4);
    vec3 warm = vec3(0.5, 0.25, 0.1);
    vec3 baseColor = mix(cold, warm, temperature_norm);
    
    // Luminosità: elevazione
    baseColor *= 0.5 + ele_norm * 0.5;
    
    // Nebbia: nuvole + visibilità
    float fogDensity = cloud_norm * (1.0 - visibility_norm);
    vec3 fogColor = vec3(0.6, 0.6, 0.65);
    baseColor = mix(baseColor, fogColor, fogDensity * 0.6);
    
    // Turbolenza: vento
    float turbulence = wind_norm * 0.15;
    uv += turbulence * sin(uv.yx * 10.0 + iTime);
    
    // Rumore pioggia
    if (precip_norm > 0.01) {
        float rain = fract(sin(dot(uv * 100.0, vec2(12.9898, 78.233))) * 43758.5453);
        baseColor += vec3(rain * precip_norm * 0.1);
    }
    
    fragColor = vec4(baseColor, 1.0);
}
